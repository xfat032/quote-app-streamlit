# 更新日志

## v1.0.9｜2026-05-12

- 修复异常 docx 读取：当 Word 文档存在 `Target="../NULL"` 等坏图片关系时，文件读取模块会回退到 `word/document.xml` 提取正文文本，不再直接报错。
- 修复方案扫描中孤立命中的未归属项：孤立的帐篷摊位归入公共“其他搭建类”，文创礼品、活动贴纸、NPC互动服务、节目演出、赛事活动归入“人员类及其他”，合作品牌联合宣传归入“活动宣传”。
- 调整本地验证程序：低于 500 字的短大纲/残片按警告处理，不再和完整方案的切片异常混为同一类失败。
- 增加下载文件回归测试，覆盖坏 docx 兜底读取和孤立显式命中项归属。

## v1.0.8｜2026-05-12

- 按用户最新要求更新 `AGENTS.md` 永久规则，补充网页端项目定位、部署链路保护、UI 保护、报价单字段、Excel 导出数据源、活动板块来源、报价项来源、建议补充项、成本价格和文件职责约束。
- 更新当前版本提示，明确后续任务需遵守新版 Agent 规则。

## v1.0.7｜2026-05-12

- 新增 `scripts/validate_app_experience.py` 本地验证程序，统一执行语法检查、核心回归测试、Streamlit AppTest 交互链路、真实 Streamlit 服务启动检查。
- 验证程序支持扫描下载目录或指定目录中的 `.txt/.docx/.pdf` 方案文件，输出 main 板块数量、报价项数量、未归属数量和活动内容切片异常。
- 验证程序会生成 `quote_app/output/internal_validation_*.json` 报告，便于复盘内部测试 bug 和流畅度指标。
- 补充真实活动标题识别词，恢复 `启动礼`、`会员互动`、`绿色行动周` 等样本活动标题的 main 板块识别。

## v1.0.6｜2026-05-11

- 收紧活动板块 main 识别：过滤活动主题、活动口号、主题解读、活动目的、宏观背景、微观痛点、活动效益预估、活动整体总图等非活动内容标题。
- 过滤 PDF/表格碎片和日期乱码：`0X`、`3 4`、`... ... ...`、日期范围、地点+室内/户外、断行清单不再进入 main 板块。
- 过滤物料、视觉、排期和执行说明类标题：道旗、手举牌、活动导览、活动铺排表、项目进度时间表、作品征集/运输/维护环节、人员配置组别等不再作为活动板块。
- 排除超短内联 `活动内容：...` 摘要切片，避免 PDF 目录或概述页的 76 字摘要被误当作稳定活动内容区域。
- 收紧报价项上下文：背景说明中的非遗演绎、特色美食、活动地图、每一缕灯光、记忆印章、封面游园会等不再直接生成正式报价项。
- 补充导出前归属兜底：测试文件中原先出现的未归属项目已归零，公共物料和近邻板块可正确归入人工确认链路。
- 增加下载文件 bug 回归测试，覆盖 main 过宽、PDF 碎片、物料视觉误判、短摘要切片和未归属项。
