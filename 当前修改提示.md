# 当前修改提示

## 当前版本

v1.0.9

## 当前版本状态

项目已完成基础模块化拆分，并已修复建议补充项、未识别候选处理区的普通表格编辑刷新风险。当前已针对海口非遗主题活动方案修复活动内容切片过长、宣传复盘误入主活动板块、泛板块误入报价区和部分误命中问题。当前已要求 `内容/尺寸/工艺` 只展示项目内容描述，确认/核价类描述必须留在备注或确认状态中。当前已收紧 `餐饮体验`、`茶歇服务` 的弱词命中，`老爸茶器具`、`茶点礼盒`、`点心制作体验` 等文化/文创/产品语境不能直接生成餐饮报价项。当前已进一步收紧 main 活动板块识别，过滤活动主题、主题解读、背景、排期、物料视觉、地点区域、执行环节和 PDF 表格碎片；超短内联 `活动内容：...` 摘要不再作为稳定活动内容切片。当前新增本地体验验证程序 `scripts/validate_app_experience.py`，用于日常运行语法检查、核心回归、Streamlit 交互链路、服务启动检查和下载目录方案扫描。当前已修复坏 docx 图片关系导致的读取失败、部分孤立显式命中项未归属，以及短大纲残片在验证中误报为完整方案失败的问题。当前 `AGENTS.md` 已按用户最新要求更新，后续任务必须遵守网页端项目定位、部署链路保护、UI 保护、报价单字段、Excel 导出数据源、活动板块来源、报价项来源、建议补充项和成本价格约束。后续每次代码修改完成并本地验证通过后，默认同步到线上 Streamlit 项目。当前重点是保持识别链路、人工确认链路、最终报价单编辑、Excel 导出链路、体验验证链路和线上部署链路稳定。

## 当前核心功能

- 支持粘贴或上传活动方案文本。
- 支持活动板块识别。
- 支持报价项识别。
- 支持活动板块人工确认。
- 支持报价项保留、编辑、排除。
- 支持建议补充项人工勾选加入报价单。
- 支持最终报价单编辑。
- 支持按最终报价单导出 Excel。
- 支持活动内容区域和活动板块候选调试信息。
- 支持通过 `scripts/validate_app_experience.py` 进行本地流畅度、用户体验链路和内部 bug 回归验证。
- 支持文旅活动、节庆活动、展览展陈、阅读活动、团建、培训等方案类型的基础识别。
- 建议补充项和未识别候选处理区需通过按钮提交后才应用，普通编辑不得触发业务更新。
- 活动内容区域遇到现场空间规划、宣传推广、执行安排、项目成效等后续章节时应停止作为主活动板块来源。
- 已有明确活动板块时，不得用市集活动、互动体验、阅读活动等泛推断板块覆盖最终报价板块。
- `内容/尺寸/工艺` 列不得出现 `需确认`、`按...确认`、`按...核价`、`核定`、`报价口径` 等确认/核价类描述；如方案无尺寸规格，只写项目内容描述。
- `餐饮体验` 和 `茶歇服务` 不能只因出现 `老爸茶`、`茶点`、`美食摊位` 等弱词就进入正式报价单；必须有餐饮服务、食品留样、提供/供应/售卖餐饮、餐券、餐标、用餐/就餐、茶歇服务等明确执行语境。
- 活动主题、活动口号、主题解读、活动目的、宏观背景、微观痛点、调研发现、活动效益预估、活动整体总图等元信息不得进入 main 活动板块。
- 道旗、手举牌、活动导览、活动铺排表、项目进度时间表、活动区域示意、美陈 DP、镜月舞台、作品征集/运输/维护环节、人员配置组别等物料、排期、区域或执行说明不得进入 main 活动板块。
- PDF 抽取出的日期乱码、表格碎片、地点+室内/户外、断行清单和超短内联 `活动内容：...` 摘要不得作为稳定活动板块来源。
- 修改完成并通过本地验证后，默认提交并推送到 GitHub 远端 `origin/main`，用于触发 Streamlit Cloud 线上部署。
- 如果线上同步失败，最终回复必须说明失败原因、已完成的本地验证结果和当前线上版本未更新的事实。

## 当前临时限制

- 不接 AI。
- 不填真实价格。
- 不改 Excel 版式，除非明确要求。
- 不绕过人工确认。
- 不用中间识别结果覆盖用户最终编辑结果。

## 当前修改前必须执行

1. 读取 `AGENTS.md`。
2. 读取 `更新日志.md`。
3. 读取 `当前修改提示.md`。
4. 执行完整项目 zip 备份。
5. 检查 Git 工作区状态。
6. 判断本次修改是否会影响核心链路、人工确认、最终报价单编辑或 Excel 导出。
7. 修改完成并验证通过后，同步线上部署；如无法同步，明确说明卡点。
