# AGENTS.md｜活动方案报价单生成器项目规则

## 1. 每次修改前必须先读

开始任何修改前，必须先读取：

1. `更新日志.md`
2. `当前修改提示.md`

如果这两个文件与旧记忆、旧指令、旧代码注释冲突，以这两个文件为准。

`AGENTS.md` 是永久规则，不写临时需求、不写当前版本细节、不写会频繁变化的报价规则。

## 2. 每次修改前必须备份

每次修改代码前，必须先备份当前项目完整版本。

备份位置：

`backups/`

备份格式：

`backup_YYYYMMDD_HHMMSS_修改前.zip`

备份时排除：

`.venv/`
`.git/`
`backups/`
`output/`
`__pycache__/`
`.DS_Store`
`node_modules/`
`.next/`
`dist/`
`build/`

备份失败时，必须停止修改。

## 3. 项目目标不能改

本项目是一个已经上传到网页端使用的活动方案报价单生成器。

固定流程：

方案输入 / 文件上传
→ 活动板块识别
→ 报价项识别
→ 人工确认
→ 最终报价单编辑
→ Excel 导出

不能改成：

AI聊天工具
ERP
合同系统
客户管理系统
复杂后台系统
纯本地工具
纯命令行工具

除非用户明确要求。

## 4. 部署链路不能乱动

这是已上线网页项目。

除非用户明确要求，否则禁止修改：

部署平台配置
构建命令
启动命令
环境变量读取方式
域名 / 路由 / 访问路径
前后端通信方式
文件上传流程
Excel 下载流程

如果必须修改部署相关文件，必须先说明影响范围。

## 5. 可以改什么

可以修改：

1. 活动板块识别规则
2. 报价项识别规则
3. 活动板块与报价项的归属逻辑
4. 未归属板块处理逻辑
5. 建议补充项逻辑
6. 规则库读取逻辑
7. 成本库 / 价格库匹配逻辑
8. 数量、面积、规格提取逻辑
9. 文件读取兼容性
10. 页面交互体验问题
11. 表格不刷新、不跳顶的问题
12. Excel 导出数据准确性
13. 回归测试脚本
14. 错误提示和调试信息

## 6. 不可以改什么

除非用户明确要求，否则禁止修改：

1. 项目整体技术栈
2. 网页端部署方式
3. 页面主流程
4. Excel 报价单基础版式
5. 最终报价单字段结构
6. 人工确认流程
7. 最终报价单编辑区
8. `final_quote_df` 作为导出唯一数据源的原则
9. 备份机制
10. 更新日志与当前修改提示机制
11. `AGENTS.md` 本身的永久规则定位

禁止擅自接入：

OpenAI API
Kimi API
Gemini API
Moonshot API
任何 AI API

禁止编造价格。

没有价格时，单价保持 `0` 或空。

## 7. UI 不可破坏规则

页面主流程必须保留：

1. 方案输入区
2. 识别结果摘要
3. 活动板块确认区
4. 报价项保留与确认
5. 建议补充项
6. 最终报价单编辑区
7. 导出 Excel
8. 高级调试信息

不要重做 UI。
不要删除人工确认区。
不要把调试信息放到主界面。
不要让普通表格编辑后页面跳回顶部。

## 8. 报价单数据规则

最终报价单字段固定为：

项目分类
项目
内容/尺寸/工艺
数量
单位
单价
合计
备注

其中：

`项目分类 = quote_section`

正式报价单不能显示：

`activity_type`
`internal_category`
`quote_section_order`
`source_status`
`confirm_status`
`原始命中词`
`调试字段`

这些只能放到高级调试信息里。

## 9. Excel 导出规则

Excel 导出的唯一数据源必须是：

`final_quote_df`

禁止导出时重新从以下数据生成：

`raw_quote_df`
`working_quote_df`
`rules_config.json`
`quote_items`
`activity_type`

用户在最终报价单编辑区改过的项目分类、项目名称、数量、单位、单价、备注，必须原样导出。

Excel 表头固定：

序号
项目分类
项目
内容/尺寸/工艺
数量
单位
单价
合计（元）
备注

不要随意改 Excel 版式。

## 10. 活动板块规则

最终报价单按：

活动宣传
具体活动板块
美陈搭建类
其他搭建类
未归属板块
人员类及其他

排序。

`人员类及其他` 固定最后。

活动板块必须来自当前方案原文，不能凭历史规则、白名单、模板或活动类型凭空生成。

活动类型只用于内部判断，不能作为最终项目分类。

## 11. 报价项规则

报价项进入最终报价单必须有依据。

允许进入最终报价单的来源：

`explicit_text`
`module_completion`
`user_selected_suggestion`

不能直接进入最终报价单的来源：

`activity_type_suggestion`
`template_suggestion`
`weak_guess`
`generic_completion`
`unknown`

这些只能进建议补充项，用户勾选后才能加入报价单。

## 12. 建议补充项规则

建议补充项默认不进入报价单。

只有用户勾选并点击“应用建议补充项”后，才能写入最终报价单。

未勾选的建议补充项不能导出到 Excel。

## 13. 成本和价格规则

价格只能来自：

价格库
成本库
用户手动填写

禁止程序自行编价格。

底层成本库优先按材料和工艺计价，例如：

桁架+喷绘 / ㎡
KT板 / ㎡
PVC板 / ㎡
帐篷租赁 / 个
桌椅租赁 / 套
执行人员 / 人
运输费 / 趟

主题展板、地图墙、签到墙默认规格：

4m × 3m
12㎡
桁架+喷绘
按面积计价

## 14. 文件职责

不要把所有逻辑堆进单个入口文件。

应保持：

页面入口：界面与流程串联
文件读取模块：文件上传、文本提取
活动类型模块：活动类型识别
报价项识别模块：关键词、规则、模块补全
报价构建模块：活动板块、quote_section、报价行构建
Excel 导出模块：报价单导出
规则库：规则和同义词
价格库 / 成本库：价格与成本数据
