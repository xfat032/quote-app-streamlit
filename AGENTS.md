# AGENTS.md｜项目永久规则

## 第一条：每次修改前必须先读版本文件

每次开始修改代码前，必须先读取并遵守以下两个文件：

1. `更新日志.md`
2. `当前修改提示.md`

这两个文件记录当前版本规则、已完成修改、正在处理的问题、临时限制和最新决策。

如果这两个文件中的规则与历史记忆、旧指令、旧代码注释冲突，以这两个文件为准。

## 第二条：每次修改前必须备份当前版本

每次开始修改代码前，必须先备份当前项目完整版本。

备份要求：

1. 备份整个项目目录，不只备份单个文件。
2. 备份为 `.zip` 压缩包。
3. 备份文件放入项目根目录下的 `backups/` 文件夹。
4. 如果 `backups/` 不存在，先创建。
5. 备份文件名格式：

backup_YYYYMMDD_HHMMSS_修改前.zip

示例：

backups/backup_20260510_153000_修改前.zip

6. 备份时必须排除以下内容：

.venv/
__pycache__/
.DS_Store
.git/
backups/
output/

7. 备份完成后，才能开始修改代码。
8. 如果备份失败，必须停止修改并说明原因。

可使用以下命令备份：

```bash
mkdir -p backups && zip -r "backups/backup_$(date +%Y%m%d_%H%M%S)_修改前.zip" . \
  -x ".venv/*" "*/.venv/*" "__pycache__/*" "*/__pycache__/*" "*.pyc" \
     ".DS_Store" "*/.DS_Store" ".git/*" "backups/*" "output/*" "*/output/*"
```

## AGENTS.md 的定位

`AGENTS.md` 是项目永久规则文件。

它只记录任何版本都不能破坏的底层约束。

不要把临时需求、当前版本规则、报价分类规则、识别规则、UI细节、价格规则写进本文件。

这些内容应写入：

- `更新日志.md`
- `当前修改提示.md`

## 永久不能破坏的项目链路

本项目核心链路固定为：

方案输入  
→ 活动板块识别  
→ 报价项识别  
→ 人工确认  
→ 最终报价单编辑  
→ Excel 导出

任何修改都不能跳过“人工确认”和“最终报价单编辑”。

## 永久不能破坏的数据原则

最终 Excel 导出必须以用户最终编辑后的报价单数据为准。

不能绕过最终编辑区，重新用原始识别结果、规则库或中间数据生成 Excel。

## 永久不能破坏的人工确认原则

自动识别不能被视为最终结果。

活动板块、报价项、建议补充项、未归属内容，都必须允许用户人工确认、修改或排除。

## 永久不能破坏的文件职责

不要把所有逻辑堆进 `app.py`。

文件职责应保持清晰：

- `app.py`：界面与流程串联
- `app_config.py`：应用路径与界面表格常量
- `app_services.py`：页面状态、表格转换、人工确认和导出前数据服务
- `core/text_reader.py`：文件读取
- `core/activity_classifier.py`：活动类型识别
- `core/extractor.py`：报价项识别
- `core/section_recognition.py`：活动板块、子活动、报价板块识别
- `core/quote_builder.py`：报价行构建、价格库读取、去重和导出前报价数据整理
- `core/quote_constants.py`：报价和活动识别常量
- `core/quote_categories.py`：报价分类归一与排序
- `core/quote_utils.py`：报价构建通用工具函数
- `core/excel_exporter.py`：Excel 导出
- `data/rules_config.json`：规则库
- `data/price_db.xlsx`：价格库或成本库

## 永久不能破坏的交互原则

普通表格编辑不能导致页面频繁刷新、跳回顶部或丢失用户已编辑内容。

用户编辑过的数据不得被自动识别结果覆盖。

## 修改前检查

每次修改前必须先确认：

1. 是否已读取 `更新日志.md`
2. 是否已读取 `当前修改提示.md`
3. 是否已完成当前版本 zip 备份
4. 是否会破坏核心链路
5. 是否会覆盖用户编辑结果
6. 是否会绕过人工确认
7. 是否会让导出结果和页面最终报价单不一致

## 禁止行为

禁止在未读取 `更新日志.md` 和 `当前修改提示.md` 的情况下直接修改代码。

禁止未备份当前版本就开始修改代码。

禁止删除人工确认流程。

禁止绕过最终报价单编辑区导出 Excel。

禁止让中间识别结果覆盖用户最终编辑结果。

禁止把临时版本规则写进 `AGENTS.md`。
